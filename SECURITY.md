# Security Policy / Política de Segurança (v1.0)

Este documento descreve a política de segurança e os limites éticos do sistema CTF Professor.
This document outlines the security policy and ethical boundaries of the CTF Professor system.

---

## 🇧🇷 Português Brasileiro

### 🛡️ Guardrails de Segurança (Limites Éticos)

O CTF Professor foi projetado exclusivamente para fins educacionais. Os seguintes limites são rigorosamente aplicados:

1.  **Não-Agressividade**: O sistema nunca deve ser usado para atacar alvos do mundo real ou sistemas sem autorização explícita.
2.  **Ambiente Controlado**: O uso de ferramentas de exploração (`pwntools`, `metasploit`, etc.) deve ser feito preferencialmente em redes isoladas ou sandboxes (veja `malware-sandboxing`).
3.  **Privacidade**: O sistema não coleta nem armazena dados sensíveis dos desafios analisados além do escopo da sessão.
4.  **Educação sobre Exploração**: O foco é sempre a compreensão da vulnerabilidade. O sistema desencoraja o uso de "scripts prontos" sem entendimento.

### 🚩 Reportando Vulnerabilidades

Se você encontrar um problema de segurança neste projeto ou no comportamento dos agentes que possa levar a usos maliciosos, por favor, entre em contato diretamente.

## 🇺🇸 English

### 🛡️ Security Guardrails (Ethical Boundaries)

CTF Professor is designed strictly for educational purposes. The following boundaries are strictly enforced:

1.  **Non-Aggression**: The system must never be used to attack real-world targets or systems without explicit authorization.
2.  **Controlled Environment**: Use of exploitation tools (`pwntools`, `metasploit`, etc.) should preferably be done in isolated networks or sandboxes (see `malware-sandboxing`).
3.  **Privacy**: The system does not collect or store sensitive data from the analyzed challenges outside the session scope.
4.  **Exploitation Education**: Focus is always on understanding the vulnerability. The system discourages the use of "ready-made scripts" without understanding.

### 🚩 Reporting Vulnerabilities

If you find a security issue in this project or in the agents' behavior that could lead to malicious use, please contact us directly.
